# HeliosHash-DAO Agent Guidelines

## Project Overview
HeliosHash DAO is a decentralized autonomous organization built with Next.js 14 frontend and Internet Computer Protocol (ICP) backend canisters.

## Architecture
- **Frontend**: Next.js 14 in `apps/web/` (Pages Router)
- **Backend**: ICP canisters in `apps/backend/` 
- **Smart Contracts**: DFX-managed canisters
- **Database**: On-chain storage via ICP

## Development Setup

### Prerequisites
\`\`\`bash
# Install dependencies
pnpm install

# Start DFX local network
dfx start --background

# Create canisters (if first time)
dfx canister create --all
\`\`\`

### Key Commands
\`\`\`bash
# Frontend Development
cd apps/web && pnpm dev

# Backend Development  
cd apps/backend
dfx generate    # Generate type declarations
dfx build       # Build canisters
dfx deploy      # Deploy to local network

# From root (if workspace configured)
pnpm dev
\`\`\`

## Project Structure

### Frontend (`apps/web/`)
- Next.js 14 with Pages Router
- TypeScript, Tailwind CSS
- Source code in `src/` directory
- Built files in `out/` directory

### Backend (`apps/backend/`)
- ICP canister-based backend
- Canisters: `helioshash_backend_backend`, `helioshash_backend_frontend`
- Type declarations auto-generated by DFX
- No traditional Express.js dependencies needed

## ICP Development Workflow

### Canister Management
\`\`\`bash
# Create canisters (first time)
dfx canister create --all

# Generate type declarations
dfx generate

# Build canisters
dfx build

# Deploy to local network
dfx deploy

# Stop local network
dfx stop
\`\`\`

## Critical Files
- `apps/web/package.json` - Frontend dependencies
- `apps/backend/package.json` - Backend DFX scripts
- `apps/backend/dfx.json` - ICP project configuration
- Root `package.json` - Workspace configuration

## Agent Notes
- This is an **ICP-based application** using canisters, not traditional Express.js
- Backend dependencies are handled by DFX, not npm
- Frontend uses **Pages Router** in `apps/web/src/`
- Always run `dfx generate` after canister changes
- Development requires local DFX network running
